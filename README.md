# Career Guidance and Employment Integration Platform

A comprehensive web application platform designed to assist high school students in discovering higher learning institutions in Lesotho, applying for courses online, and facilitating career placement for graduates.

## üéØ Project Overview

This platform serves as a bridge between students, higher learning institutions, and companies, providing:
- Course discovery and application management
- Institution and course management
- Job posting and applicant matching
- Automated notifications for qualified candidates

- **Frontend**: React.js
- **Backend**: Node.js (Express)
- **Database**: Firebase (Firestore)
- **Authentication**: Firebase Auth + JWT

## üìã System Modules

### 1. Admin Module
- Login and authentication
- Add and manage higher learning institutions
- Add faculties and courses under each institution
- Delete or update institutions, faculties, and courses
- Publish admissions and monitor registered users
- Manage registered companies (approve, suspend, or delete accounts)
- View and manage system reports

### 2. Institute Module
- Register (with email verification) and login
- Add and manage faculties
- Add and manage courses
- View student applications
- Publish admissions
- Manage student status (admitted, rejected, pending)
- Update institution profile

### 3. Student Module
- Register (with email verification) and login
- Apply for courses (maximum of two per institution)
- View admissions results
- Update profile and upload additional documents
- Upload academic transcripts and additional certificates upon completion of studies
- View and apply for available job postings
- Receive notifications for new job opportunities matching their profile
- Select institution when admitted to multiple institutions

### 4. Company Module
- Register (with email verification) and login
- Post job opportunities with defined qualifications and requirements
- View filtered and qualified applicants automatically generated by the system based on:
  - Academic performance
  - Extra certificates
  - Work experience
  - Relevance to job post
- Receive only applications that are ready for interview consideration
- Update company profile

## üîí Business Rules Implemented

1. **Maximum Applications**: Students can apply for a maximum of 2 courses per institution
2. **No Duplicate Admissions**: Institutions cannot admit the same student into multiple programs
3. **Qualification Validation**: Students cannot apply for courses they do not qualify for
4. **Qualified Notifications**: Only qualified students receive job notifications from connected companies
5. **Institution Selection**: If a student is admitted to multiple institutions, they can select one, which automatically removes them from others and promotes the first student from the waiting list

## üöÄ Getting Started

### Prerequisites

- Node.js (v14 or higher)
- npm or yarn
- Firebase project with Firestore enabled
- Firebase service account credentials
- Email service credentials (Gmail or other SMTP service)

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd career-guidance-platform
   ```

2. **Backend Setup**
   ```bash
   cd backend
   npm install
   ```

3. **Frontend Setup**
   ```bash
   cd frontend
   npm install
   ```

4. **Environment Variables**

   Create a `.env` file in the `backend` directory:
   ```env
   FIREBASE_PROJECT_ID=your-project-id
   FIREBASE_PRIVATE_KEY_ID=your-private-key-id
   FIREBASE_PRIVATE_KEY="your-private-key"
   FIREBASE_CLIENT_EMAIL=your-client-email
   FIREBASE_CLIENT_ID=your-client-id
   FIREBASE_CLIENT_CERT_URL=your-cert-url
   JWT_SECRET=your-jwt-secret
   EMAIL_USER=your-email@gmail.com
   EMAIL_PASS=your-app-password
   FRONTEND_URL=http://localhost:3000
   PORT=5000
   ```

5. **Run the Application**

   **Backend:**
   ```bash
   cd backend
   npm run dev  # Development mode with nodemon
   # or
   npm start    # Production mode
   ```

   **Frontend:**
   ```bash
   cd frontend
   npm start
   ```

## üì° API Endpoints

### Authentication
- `POST /api/auth/register` - User registration
- `POST /api/auth/login` - User login
- `GET /api/auth/verify-email/:token` - Email verification

### Admin Routes
- `GET /api/admin/institutions` - Get all institutions
- `POST /api/admin/institutions` - Add institution
- `PUT /api/admin/institutions/:id` - Update institution
- `DELETE /api/admin/institutions/:id` - Delete institution
- `POST /api/admin/institutions/:id/faculties` - Add faculty
- `PUT /api/admin/institutions/:id/faculties/:facultyId` - Update faculty
- `DELETE /api/admin/institutions/:id/faculties/:facultyId` - Delete faculty
- `POST /api/admin/institutions/:id/faculties/:facultyId/courses` - Add course
- `PUT /api/admin/institutions/:id/faculties/:facultyId/courses/:courseId` - Update course
- `DELETE /api/admin/institutions/:id/faculties/:facultyId/courses/:courseId` - Delete course
- `GET /api/admin/companies` - Get all companies
- `PUT /api/admin/companies/:id/approve` - Approve company
- `PUT /api/admin/companies/:id/suspend` - Suspend company
- `DELETE /api/admin/companies/:id` - Delete company
- `GET /api/admin/reports/statistics` - Get system statistics
- `GET /api/admin/reports/users` - Get users report
- `GET /api/admin/reports/admissions` - Get admissions report

### Student Routes
- `GET /api/student/institutions` - Get all institutions with courses
- `GET /api/student/applications` - Get student's applications
- `POST /api/student/applications` - Apply for course
- `POST /api/student/transcript` - Upload transcript
- `POST /api/student/select-institution` - Select institution when admitted to multiple
- `GET /api/student/jobs` - Get available jobs
- `POST /api/student/jobs/:jobId/apply` - Apply for job
- `GET /api/student/notifications` - Get notifications
- `PUT /api/student/notifications/:id/read` - Mark notification as read

### Institute Routes
- `GET /api/institute/:instituteId/faculties` - Get faculties
- `POST /api/institute/:instituteId/faculties` - Add faculty
- `PUT /api/institute/:instituteId/faculties/:facultyId` - Update faculty
- `DELETE /api/institute/:instituteId/faculties/:facultyId` - Delete faculty
- `GET /api/institute/:instituteId/faculties/:facultyId/courses` - Get courses
- `POST /api/institute/:instituteId/faculties/:facultyId/courses` - Add course
- `PUT /api/institute/:instituteId/faculties/:facultyId/courses/:courseId` - Update course
- `DELETE /api/institute/:instituteId/faculties/:facultyId/courses/:courseId` - Delete course
- `GET /api/institute/:instituteId/applications` - Get applications
- `PUT /api/institute/:instituteId/applications/:applicationId/status` - Update application status
- `POST /api/institute/:instituteId/admissions/publish` - Publish admissions
- `GET /api/institute/:instituteId/profile` - Get profile
- `PUT /api/institute/:instituteId/profile` - Update profile

### Company Routes
- `GET /api/company/profile` - Get company profile
- `PUT /api/company/profile` - Update company profile
- `POST /api/company/jobs` - Create job posting
- `GET /api/company/jobs` - Get company's jobs
- `PUT /api/company/jobs/:jobId` - Update job
- `DELETE /api/company/jobs/:jobId` - Delete job
- `GET /api/company/jobs/:jobId/applicants` - Get qualified applicants
- `PUT /api/company/jobs/:jobId/applicants/:applicationId` - Update applicant status

## üîê Authentication

All protected routes require a Bearer token in the Authorization header:
```
Authorization: Bearer <token>
```

Tokens are obtained through the login endpoint and expire after 24 hours.

## üìù Database Structure

### Collections
- `users` - User accounts
- `students` - Student profiles and qualifications
- `institutions` - Institution data
- `institutions/{id}/faculties` - Faculties subcollection
- `institutions/{id}/faculties/{id}/courses` - Courses subcollection
- `institutions/{id}/applications` - Applications subcollection
- `companies` - Company profiles
- `applications` - Course applications
- `jobs` - Job postings
- `jobApplications` - Job applications
- `notifications` - User notifications
- `emailVerifications` - Email verification tokens
- `admissions` - Published admissions

3. Update `FRONTEND_URL` in backend environment variables
